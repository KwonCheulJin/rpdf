# ADR-003: PDF 파서 — lopdf 미사용, 직접 구현

**날짜**: 2026-04-01  
**상태**: 승인됨  
**결정자**: KwonCheulJin

## 맥락

PDF 파싱에 기존 `lopdf` 크레이트를 사용하는 것과 직접 구현하는 것 중 선택이 필요하다.

## 결정

`lopdf`를 **파싱**에 사용하지 않는다. `rpdf-parser`를 직접 구현한다.  
단, **저장(Serializer)** 구현 시 `lopdf` writer 재활용은 허용한다 (v0.3 예정).

## 근거

- 학습 목표: PDF 스펙(ISO 32000) 직접 구현으로 내부 동작 이해
- `lopdf` IR이 도메인 모델과 불일치 → 변환 레이어 복잡도 증가
- 직접 구현 시 xref 스트림, 오브젝트 스트림 등 복잡한 기능을 점진적으로 추가 가능
- DDD 원칙: PDF 스펙 용어를 코드에 그대로 반영 (`XrefTable`, `PdfTrailer` 등)

## 결과

- `rpdf-parser`: nom 없이 수작업 바이트 파서
- 스냅샷 회귀 테스트 (`insta`)로 파서 정확도 검증
- 현재 한계: 간접 참조 `/Length`, 암호화 PDF 미지원 (v0.2 이후 점진적 해소)

## 기각된 대안

- **lopdf 전면 사용**: IR 불일치, 커스텀 오류 처리 어려움
- **pdf-extract**: 텍스트 추출 특화, 구조 파싱 미지원
