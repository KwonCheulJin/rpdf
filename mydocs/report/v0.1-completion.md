## 4. v0.2 진입 포인트 — 렌더링 뼈대

v0.2의 목표는 `pdfium-render`를 도입하여 파서 출력을 시각적으로 검증하는 것이다.

### v0.2가 의지할 v0.1 IR

```rust
// v0.2 렌더링 시작점
let doc = load_document(&bytes)?;
for page in doc.pages() {
    // page.media_box() → 캔버스 크기
    // page.content() → ContentStreamOperation 시퀀스
    //   → v0.2에서 ContentElement로 해석 후 렌더링
    // page.rotation() → 회전 변환
}
```

### v0.2에서 수정 예정인 v0.1 한계

| 항목 | 상태 | 수정 우선순위 |
|------|------|------------|
| `/Length` indirect reference 미지원 | 에러 반환 | 높음 (canvas.pdf 등 파싱 실패) |
| 페이지 트리 순환 참조 스택 오버플로우 | 스택 오버플로우 | 높음 (보안) |
| UTF-16BE 메타데이터 디코딩 | 깨진 문자 | 중간 |
| `Contents` array-of-streams 비표준 | 에러 반환 | 중간 |
| fw4-2024.pdf 성능 이상치 (139ms) | 느림 | 낮음 |

### v0.2 준비 사항

1. `pdfium-render` crate 도입 검토 (docs.rs 공개 API 확인 필요)
2. `rpdf-render` crate 신규 추가 (workspace)
3. 렌더링 결과 이미지를 테스트 fixture와 비교하는 인프라

## 5. 학습 자산 — 프로세스 진화

### CLAUDE.md에 남긴 패턴들

v0.1 10개 타스크를 거치며 CLAUDE.md에 쌓인 핵심 패턴들:

- **에러 변형 도달 가능성**: 발생 코드 경로 없는 변형은 dead variant
- **외부 입력 검증 (silent failure 방지)**: wrap-around 명시적 거부
- **테스트 파일 배치**: public API → `tests/`, private → inline `#[cfg(test)]`
- **체크포인트 셀프 리뷰**: vacuous pass 방지, 정책 일관성 검사
- **자율 진행 모드 회고 채집**: 매 체크포인트별 메모 → PR 분류표

### 자율 진행 모드 평가 (Task #7~10)

| 항목 | 평가 |
|------|------|
| 트러블슈팅 즉시 작성 | Task #10에서 정착 (pages-tree 발견 즉시) |
| 회고 분류표 | Task #10에서 복원 (retro-notes.md 패턴 도입) |
| 체크포인트 보고 | A~E 각 완료 후 1~2줄 보고 정착 |
| 멈춤 조건 준수 | PR 본문 전 멈춤 준수 |
