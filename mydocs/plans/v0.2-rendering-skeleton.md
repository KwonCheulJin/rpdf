# v0.2 — 렌더링 뼈대

**마일스톤**: M020
**목표 기간**: 4~6주
**상태**: 계획
**선행 조건**: v0.1 완료

## 목표

PDF를 시각적 출력으로 변환할 수 있게 한다. 렌더링은 `pdfium-render`에 위임하되, rpdf 자체의 SVG 생성 경로도 간단히 확보한다.

## 성공 기준

1. 임의의 PDF 페이지를 PNG 이미지로 렌더링할 수 있다
2. 임의의 PDF 페이지를 SVG로 내보낼 수 있다
3. `--debug-overlay` 옵션으로 페이지·객체 경계를 시각화할 수 있다
4. 렌더링 회귀 테스트 인프라 구축 (스냅샷 이미지 비교)
5. A4 기준 1페이지 렌더링 100ms 이내 (릴리즈 빌드)

## 범위 외

- 편집 기능 (v0.3)

## 기술 노트

### SVG vs PNG의 용도 분리

- **SVG**: 디버깅, 작은 파일 크기, 고해상도 확대 가능
- **PNG**: 정확한 렌더링, 썸네일, UI 표시

웹 에디터에서는 PDFium이 PNG로 렌더링한 것을 Canvas에 표시하는 것이 기본 경로가 될 것이다.

### 디버그 오버레이 예시

```
│ p=0  rot=0  792×612          │
│                              │
│ ┌──────────┐                 │

│ │ y=400    │                 │
│ └──────────┘                 │
│                              │
│     ┌────────┐               │
│     │ img #5 │               │
│     │ y=200  │               │

└──────────────────────────────┘
```
