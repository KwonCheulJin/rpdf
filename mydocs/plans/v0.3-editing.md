# v0.3 — 편집 커맨드 개요

**마일스톤**: M030  
**선행 조건**: v0.2 완료  
**상태**: 큰 그림 확정, 상세 계획은 v0.2 완료 후 작성

---

## 핵심 가치

v0.2까지는 PDF를 "읽고 본다". v0.3부터는 "고치고 저장한다."  
`document_core` 레이어에 CQRS 패턴을 도입하고, 기본 편집 커맨드를 구현한다.  
이 단계 완료 시 **"CLI 기반 PDF 편집 도구"**가 완성된다.

---

## 예상 타스크 (Task #17–?)

| 임무 | 한 줄 설명 |
|------|-----------|
| Command/Query 트레이트 정의 | `execute`, `undo`, `name` 트레이트, `CommandEffect` |
| CommandStack 구현 | `undo_stack`, `redo_stack`, 깊이 제한 100 |
| RotatePageCommand | `/Rotate` 값 정규화, Undo 대칭성 |
| DeletePagesCommand | 페이지 트리 재구성, Undo 스냅샷 |
| MergeCommand | 리소스 이름 충돌 해결 (prefix 재작성) |
| SplitCommand / ExtractPagesCommand | 페이지 범위 명세 (`1-3,5,7-10`) |
| Serializer | `Document` → PDF 바이트. Incremental + Full rewrite 모드 |
| CLI 편집 명령 추가 | `rpdf merge/split/rotate/delete/extract` |
| 편집 회귀 테스트 | 커맨드별 Undo 대칭성 + end-to-end 저장/재파싱 |

> 타스크 번호는 v0.2 완료 후 확정.

## 핵심 의존성

- **v0.2 렌더링 완료**: 편집 결과를 시각적으로 검증해야 회귀 테스트 의미 있음
- **v0.1 IR mutable 확장**: `dirty` 플래그, `original_bytes` 추가
- **`lopdf` writer**: Serializer 구현에 재활용

---

## 성공 기준 (초안)

1. `Command`, `Query` 트레이트 정의 + 테스트 커버
2. `CommandStack` Undo/Redo 동작
3. 5개 커맨드 구현: merge, split, rotate, delete_pages, extract_pages
4. CLI로 모든 커맨드 호출 + 결과 PDF 저장 가능
5. 저장 후 재열기 시 원본 품질 보존 (pdfium 렌더링 결과 동일)

## 위험 요소

- Undo 스냅샷 메모리 사용량 (큰 PDF × 깊은 스택)
