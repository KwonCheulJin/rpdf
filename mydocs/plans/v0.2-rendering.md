# v0.2 — 렌더링 뼈대

**마일스톤**: M020  
**상태**: 계획 확정 (2026-05-04, 의존성 검증 완료)  
**선행 조건**: v0.1 완료 (Task #1–10, PR #19 머지)  
**예상 타스크**: Task #11–17 (7개)

---

## 핵심 가치

v0.1은 "파서가 PDF를 읽는다"를 증명했다.  
v0.2는 "파서 출력이 시각적으로 옳다"를 증명한다.  
렌더링 피드백 루프 없이는 파서 정확도를 눈으로 검증할 방법이 없다.

---

## 성공 기준 6가지

1. `examples/` 5개 PDF → pdfium-render로 PNG 변환 성공
2. `samples/` 28개 PNG + 이미지 스냅샷 회귀 CI 통과
3. `rpdf render <pdf> -o output.png` CLI 동작
4. `rpdf render <pdf> --svg --debug-overlay` CLI 동작
5. CI에서 pdfium 자동 설치 + 회귀 스냅샷 통과
6. v0.1 IR(`ContentStreamOperation`)이 SVG 렌더러로 시각화됨

## 마일스톤 의존성

```
v0.1 (파서 + IR)
  └── v0.2 (렌더링)          ← 지금
        └── v0.3 (편집 커맨드)
              └── v0.4 (WASM + 웹)
                    └── v0.5 (Tauri 데스크톱)
```

v0.2가 의지하는 v0.1 출력:
- `Document` IR (`pages`, `media_box`, `content`, `rotation`)
- `ContentStreamOperation` 시퀀스 (SVG 렌더러 입력)
- `samples/` 28개 PDF (회귀 입력)

## 위험 평가

| 위험 | 가능성 | 영향 | 대응 |
|------|--------|------|------|
| PDFium 동적 라이브러리 번들 복잡 | 높음 | 높음 | Task #11 분리로 위험 조기 해소. 실패 시 SVG 전용으로 v0.2 재정의 |
| macOS Gatekeeper가 다운로드 `.dylib` 차단 | 중 | 중 | `xattr -d com.apple.quarantine` 또는 notarize |
| 이미지 스냅샷 테스트 flaky (OS별 픽셀 차이) | 중 | 중 | 허용 오차 0.1% + OS별 스냅샷 분리 |
| v0.1 한계(간접 참조 `/Length`, 순환 참조)로 파싱 실패 PDF | 중 | 중 | 마주친 즉시 v0.1 수정 태스크화 |
| SVG 품질 기대치 초과 요구 | 낮음 | 낮음 | SVG는 디버그 전용. 품질 목표 낮게 명시 |
