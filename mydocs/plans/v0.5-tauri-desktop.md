# v0.5 — Tauri 데스크톱 앱 (실사용자 배포)

**마일스톤**: M050
**목표 기간**: 3~4주
**상태**: 계획
**선행 조건**: v0.4 완료

## 목표

v0.4까지 만든 웹 에디터를 Tauri 2로 감싸 데스크톱 앱으로 배포한다.
**이 단계의 성공 기준은 "여자친구가 기존 pdf-studio를 삭제하고 이 앱을 쓰는 것"이다.**

## 성공 기준

1. Windows, macOS 빌드가 성공하고 정상 실행됨
2. 유휴 상태 메모리 200MB 이하
3. 설치 파일 크기 20MB 이하
4. 기존 pdf-studio의 주요 기능(병합, 회전, 삭제, TIFF→PDF)이 모두 동작
5. 여자친구가 1주 이상 실사용하며 크래시 없음

## 범위 외

- 자동 업데이트 (v0.6)
- 코드 서명 (v0.6 또는 배포 시점에 결정)
- Linux 빌드 (선택, 필요 시)

### #33 Tauri Commands 정의

WASM API와 유사하지만 네이티브 Rust 직접 호출:

```rust
use rpdf_core::{Document, CommandStack};

#[tauri::command]
async fn open_pdf(path: String, state: tauri::State<'_, AppState>) -> Result<PdfHandle, String> {
    let bytes = tokio::fs::read(&path).await.map_err(|e| e.to_string())?;
    let doc = Document::from_bytes(&bytes).map_err(|e| e.to_string())?;
    let handle = state.register(doc);
    Ok(handle)
}

#[tauri::command]
async fn save_pdf(
    handle: PdfHandle,
    output_path: String,
    state: tauri::State<'_, AppState>,
) -> Result<(), String> { ... }

#[tauri::command]
async fn merge_pdfs(handles: Vec<PdfHandle>) -> Result<PdfHandle, String> { ... }

#[tauri::command]
async fn rotate_page(handle: PdfHandle, page: usize, degrees: i32) -> Result<(), String> { ... }

#[tauri::command]
async fn delete_pages(handle: PdfHandle, indices: Vec<usize>) -> Result<(), String> { ... }

#[tauri::command]
async fn tiff_to_pdf(tiff_paths: Vec<String>, output_path: String) -> Result<(), String> { ... }

#[tauri::command]
async fn get_thumbnail(handle: PdfHandle, page: usize, size: u32) -> Result<Vec<u8>, String> { ... }
```

### #34 프론트엔드에서 WASM 분기

기존 `wasm-bridge.ts`를 환경별 분기로 수정:

```typescript
// core/bridge.ts
const isTauri = typeof window !== 'undefined' && '__TAURI__' in window;

export const bridge = isTauri
    ? await import('./tauri-bridge').then(m => m.default)
    : await import('./wasm-bridge').then(m => m.default);
```

**Tauri 브리지**: `invoke('open_pdf', { path })`
**WASM 브리지**: `new PdfDocument(bytes)`

### #36 TIFF → PDF 변환 (기존 pdf-studio 기능 이식)

기존 pdf-studio는 Node `sharp`에 의존했지만, rpdf는 Rust `image` 크레이트 사용:

```rust
use image::io::Reader;
use lopdf::{Document, Object};

pub fn tiff_to_pdf(tiff_paths: &[PathBuf]) -> Result<Document, RpdfError> {
    let mut doc = Document::with_version("1.7");
    for path in tiff_paths {
        let img = Reader::open(path)?.decode()?;
        // JPEG 변환 후 PDF 페이지로 삽입
        add_image_page(&mut doc, &img)?;
    }
    Ok(doc)
}
```

## 프론트엔드 재사용 전략

v0.4의 `rpdf-studio`와 v0.5의 `rpdf-desktop/src/`는 거의 동일한 React 코드다. 다음 방식 중 선택:

### 옵션 A: 심볼릭 링크
- `rpdf-desktop/src` → `../rpdf-studio/src` 링크
- 장점: 즉시 동기화
- 단점: Windows 호환성 문제

### 옵션 B: 모노레포 패키지
- `@rpdf/ui` 공유 패키지로 추출
- 장점: 깔끔
- 단점: 초기 세팅 부담

### 옵션 C: 코드 복사 후 수동 동기화
- 장점: 단순
- 단점: drift 발생

**선택**: 옵션 B. pnpm workspace로 `@rpdf/ui` 공유 패키지.

```
rpdf/
├── packages/
│   ├── core/              # Rust 코어 (WASM 배포용)
│   ├── ui/                # @rpdf/ui (React 컴포넌트, 훅)
│   ├── studio/            # 웹 에디터 (ui 사용)
│   └── desktop/           # Tauri 앱 (ui 사용)
├── crates/
│   └── rpdf-core/         # Rust 크레이트 (core와 desktop에서 공유)
└── package.json
```
