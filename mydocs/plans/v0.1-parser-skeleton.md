# v0.1 — 파서 뼈대

**마일스톤**: M010
**목표 기간**: 4~6주
**상태**: 계획
**작성일**: 2026-04-24

## 목표

PDF 파일을 읽고, 구조를 이해하고, 디버깅 가능한 상태로 만든다.
**이 단계의 결과물은 "편집 가능한 PDF 편집기"가 아니라 "PDF를 이해하는 툴체인"이다.**

## 성공 기준

1. 임의의 PDF 파일을 열고 기본 메타데이터를 추출할 수 있다
2. 페이지 트리를 순회하며 각 페이지의 기본 정보를 얻을 수 있다
3. content stream을 파싱하여 연산자 시퀀스로 표현할 수 있다
4. CLI 디버깅 도구가 동작한다 (`rpdf info`, `rpdf dump`)
5. 200+ 단위 테스트 통과
6. 샘플 PDF 30개 수집 및 회귀 테스트 인프라 구축

## 범위 외 (Out of scope)

- 렌더링 (v0.2)
- 편집 (v0.3)
- UI (v0.4 이후)
- 암호화된 PDF 복호화 (v0.3 이후 검토)

#### Task #3 시작 시 ParseError 보강 메모 (Task #2 셀프 리뷰 산출)

- `XrefStreamUnsupported { xref_offset: u64 }` — Task #3 본체에서 xref stream 오프셋을 바로 활용하기 위해 필드 추가 [우선순위 높음]

- `reject_empty_input` 테스트 이름 중복 정리: `eof_tests::reject_empty_input_for_eof`, `header_tests::reject_empty_input_for_header` [낮음]
- `MissingEof` / `MissingStartXref` / `MissingTrailer`에 `searched_range` 추가 [후순위, 여유 시]

## 위험 요소

| 위험 | 가능성 | 영향 | 대응 |
| --- | --- | --- | --- |
| `lopdf`가 일부 PDF를 파싱 못 함 | 중 | 중 | 문제 PDF는 `mupdf-rs` 등으로 변환 전처리 |
| 샘플 PDF 수집에 시간 오래 걸림 | 중 | 낮음 | 공개 데이터셋 활용 (govdocs1 등) |
| xref 복구가 복잡해짐 | 중 | 높음 | v0.1에서는 깨진 파일 지원 제외, 에러 반환만 |

## 테스트 전략

### 단위 테스트
- 각 파서 모듈마다 최소 10개 테스트
- 경계 케이스 (빈 딕셔너리, 중첩 배열, 큰 스트림) 포함

### 속성 기반 테스트 (proptest)
- 임의의 바이트 입력에 대해 크래시하지 않음
- 파싱 결과를 다시 직렬화했을 때 의미 동등

### 회귀 테스트
- 수집한 30개 샘플 각각에 대해 `rpdf info` 출력 스냅샷
- CI에서 매 커밋 실행
